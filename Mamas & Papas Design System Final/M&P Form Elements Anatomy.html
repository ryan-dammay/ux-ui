Do
Use Primary only once per page — reserve it for the top-level CTA.
Always include an icon on Primary CTAs to aid scannability.
Add aria-label to all icon-only buttons.
Show a loading spinner when a form is submitting.
Keep button text concise — 1–3 words maximum.
Don't
Don't use more than one Primary button per view.
Don't disable submit without explaining why (form validation).
Don't use icon-only buttons for primary actions without labels.
Don't use Link buttons for destructive actions — use a red-tinted Ghost instead.
Don't mix button styles for the same level of action hierarchy.

Usage Guidelines

Do
Always use a visible label above every field — never rely on placeholder alone.
Show helper text proactively for complex fields (e.g. password requirements).
Always set aria-label on search inputs and icon-only controls.
Use the correct input type (email, tel, number) for better mobile keyboards.
Show inline validation as soon as the field is blurred (on blur, not on submit).
Keep form fields left-aligned and stacked vertically on mobile.
Use Toggle for immediate-effect settings; Checkbox for form submission consent.
Ensure all inputs have a min touch target height of 44px on mobile.
Don't
Don't use placeholder text as a label — it disappears on input and fails accessibility.
Don't show error messages before the user has interacted with the field.
Don't disable a submit button without explaining why the form is incomplete.
Don't use red colour alone to indicate errors — always include an icon and text.
Don't allow horizontal resize on textareas — it breaks the layout grid.
Don't use a Radio when only one option is available — use a Checkbox instead.
Don't use custom font-sizes below 16px on mobile inputs — it triggers iOS zoom.
Don't mix Toggle and Checkbox styles for the same type of action.
Accessibility Checklist
Every input has an associated <label>
Error messages are linked via aria-describedby
Required fields have aria-required="true"
Error state uses aria-invalid="true"
Focus ring visible on all interactive elements
Color contrast ≥ 4.5:1 for all text (WCAG AA)
Keyboard navigable with Tab / Shift+Tab
Touch targets ≥ 44×44px on mobile
