2 Add Colour Variables — Token Format
💡
Token naming convention Use dot notation with category prefix — color.primary, text.secondary. In Figma, replace dots with slashes: color/primary. This maps directly to CSS custom properties and design tokens (Style Dictionary, Tokens Studio).
⚠️
Define as tokens, not hex values. Never hardcode #198669 in a component. Always reference the token — color.primary. This way swapping a brand colour updates every component instantly.

How tokens reference each other token → token → component
💡
3-layer architecture: Core defines raw values → Semantic maps intent → Component references semantic. Never skip a layer.
// Layer 1: Core (raw values) color.primary = #198669 color.background = #FFFFFF // Layer 2: Semantic (maps intent to core) color.success = {color.primary} ↳ references core text.brand = {color.primary} ↳ references core // Layer 3: Component (uses semantic) Button/Primary/Fill = {color.primary} Button/Primary/Label = {text.on-dark} Input/Border/Focus = {color.primary} Input/Text = {text.primary} Input/Hint/Error = {text.error}

How to use breakpoints in Figma
  • 1.Create a frame → set width to the breakpoint value (e.g. 1024 for LG).
  • 2.Apply the matching Grid style — Grid/Desktop for LG+, Grid/Tablet for MD, Grid/Mobile for SM/XS.
  • 3.Name frames with a breakpoint prefix: Mobile/Home, Desktop/PDP.
  • 4.Use Auto Layout with Fill container widths so components stretch naturally across viewports.

5 Create Colour Styles
💡
Why both Variables AND Styles? Variables power component fills/strokes (linked, switchable). Styles show in the colour picker and are easier for designers to apply quickly. Keep both in sync.
⚠️
Link styles to variables When creating a colour style, set the fill value by selecting the corresponding variable. This way changing the variable updates the style too.
Core group
// Token = value color.primary = #198669 color.primary-dark= #136B52 color.secondary = #C4574A color.navy = #2C3E5A color.ocean = #175A7B color.amber = #E8A020 color.purple = #7B5EA7
Text group
// Token = value text.primary = #323232 text.secondary = #575656 text.muted = #878686 text.disabled = #D1D1D1 text.on-dark = #FFFFFF text.brand = {color.primary} text.error = {color.error}
Background group
// Token = value color.background = #FFFFFF color.surface = #F8F6F0 color.surface-alt = #F8F8F8 color.success-bg = #D8EEE8 color.error-bg = #FDA8A4
Border group
// Token = value color.border = #D1D1D1 color.border-focus= {color.primary} color.border-error= {color.error}

10 Naming Conventions & Rules
✅ Do
  • ✓Use . dot notation: color.primary, text.secondary
  • ✓In Figma use / slashes: color/primary
  • ✓Use lowercase kebab: color.primary-dark
  • ✓Describe intent, not appearance: color.error not color.red
  • ✓Reference semantic tokens in components: {color.primary}
  • ✓Keep spacing multiples of 8px
❌ Don't
  • ✗Never hardcode hex values in components — always use {token}
  • ✗Don't name by colour: color.green, color.red
  • ✗Don't skip layers — use semantic in components, not core
  • ✗Don't put all tokens in one flat collection
  • ✗Don't use event colours (#323232 Black Friday) in UI chrome
  • ✗Don't use spacing values outside the scale
